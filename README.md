## Исходные данные
В проекте используется датасет отзывов клиентов Ozon Bank: 
[Russian Bank Customer Reviews Dataset на Kaggle](https://www.kaggle.com/datasets/scorpy12/russian-bank-customer-reviews-dataset).


### Описание проекта :
Задача: автоматизировать поиск жалоб на мошенничество, кражу денег и фрод по текстовым отзывам клиентов Ozon банка. 
Основная сложность: отсутствие исходной целевой переменной и сильный дисбаланс классов (фрода < 2%). 

Я дистиллировала знания NLI-модели на базе RuBERT в CatBoostClassifier для экономии финансов и времени.

---

### Стек технологий :
* **Язык**: Python
* **NLP & Transformers**: Hugging Face `transformers` (`pipeline`), `RuBERT` (`cointegrated/rubert-base-cased-nli-twoway`)
* **Машинное обучение**: `CatBoostClassifier` (обучение на GPU T4)
* **Анализ данных и статистика**: `Pandas`, `NumPy`, `SciPy` (`chi2_contingency`)
* **Визуализация**: `Plotly Express`, `Seaborn`, `Matplotlib`
* **Валидация**: `Scikit-learn` (`f1_score`, `precision_recall_curve`, `auc`, `confusion_matrix`, `cohen_kappa_score`)

---

### Архитектурная часть решения :

1.  Вместо ручной разметки я применила NLI-модель (Zero-shot классификатор на базе RuBERT). Она выступила в роли псевдоразметчика и сгенерировала качественные дометки для целевой переменной is_fraud. Каппа Коэна на случайной выборке в 100 строк составила **1.0**, подтвердив идеальное понимание контекста трансформером.
2. NLI модели медленные (один ответ занимает от до нескольки секунд) и требуют дорогой инфраструктуры (видеокарт nvidea). CatBoost выдает ответ за несколько миллисекунд и работает на CPU. Я решила обучить CatBoostClassifier имитировать логику трансформера, используя простые фичи
3. В ходе разведочного анализа данных критерием хи-квадрат я доказала нелинейную связь между длиной текста и оценкой (`Grade`). На основе этого я добавила признак grade_text_len_ratio = text_len * (6 - Grade), которая сыграла большую роль в обучениии модели
4. Доля фрода в тесте составила всего 2.1% (95 кейсов из 4439). Проблема решена с помощью class_weights=[1, 30].

---

### Анализ результатов и метрики :

Для оценки качества я использовала отложенный test_df. Из-за дисбаланса классов выбрала метрики **PR-AUC** и **F1-Score**.
Стандартный порог 0.5 неэффективен для редких классов. Перебором вероятностей `predict_proba` я нашла лучший порог - **0.20**, который увеличил метрику F1-Score до **0.2805** (PR-AUC = **0.2531**).

#### Анализ матрицы ошибок :
* Модель успешно выудила из потока текстов почти половину скрытого фрода (46 из 95).
* Объем ложных срабатываний минимален (187).
* Часть фрода пропущена т.к. бустингу тяжело на 100% повторить логику трансформера, используя базовый мешок слов на небольшой выборке.

**Итог:** Модель  подходит как легковесный и бесплатный по железу первичный антифрод-фильтр для работы в реальном времени.
